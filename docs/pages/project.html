💬 YouTube Comment Sentiment Analysis

Research Project · NLP | 2025

Project Overview

Analyzed viewer comment data across English-learning YouTube channels to understand what draws ESL audiences and whether the content supports language acquisition.

Technical Highlights

  • Collected and processed comments from 305+ videos via the YouTube Data API, extracting structured fields into a per-channel dataframe.
  • Built an automated batch submission pipeline using the OpenAI Batch API — chunked comments into 400-comment batches, controlled concurrency with MAX_CONCURRENT = 1 to stay within token limits, and tracked job status via a tracking_log.csv.
  • Implemented failure recovery: failed jobs can be re-queued by flipping their log status from fail → pending and re-running the script, with no manual re-work required.
  • Applied bigram frequency analysis after stop-word removal to surface recurring viewer themes per channel.
  • Used multi-label classification (OpenAI API, ~20 label categories) to categorize comments into thematic clusters; aggregated results using like-weighting (log transform) and normalization.
  • Scored comments along 4 learning dimensions using keyword stem matching: Listening, Speaking/Pronunciation, Vocabulary/Grammar, and Confidence/Motivation.

🧪 Multimodal Embeddings for Molecular Property Prediction

Graduate Research Project · NLP × Computational Chemistry | 2024–2025
Sponsored by Prof. Zois Boukouvalas · Manuscript in preparation

Project Overview

Contributed to a graduate research project investigating whether fusing natural language representations with chemical structure embeddings improves molecular property prediction for drug screening applications.

Technical Highlights

  • Textual embedding pipeline: Prompted Llama-3.1-8B to generate per-molecule pharmacological descriptions from SMILES strings and molecular metrics, then encoded outputs into dense vectors using BGE-M3.
  • Prompt engineering: Designed and iterated on a system prompt constraining the LLM to medicinal chemistry analysis — balancing scientific precision with output consistency across 1,500+ molecules.
  • Data preparation: Canonicalized SMILES representations using RDKit and structured the dataset for downstream embedding and regression tasks.
  • Pipeline integration: Coordinated parallel textual and structural feature generation into a unified input for the regression model.

🎮 Pokémon AI Agent

Team Project · Computer Vision & Reinforcement Learning | 2025

Project Overview

Built an AI agent that plays Pokémon Gold (Game Boy Color, 1999) by interpreting visual input from a Game Boy emulator and making real-time decisions — no access to the game's internal state, only screen pixels. The project explored two independent methods: a CNN-based imitation learning agent and a grid-map navigation agent using Q-learning.

Technical Highlight 1 — CNN Imitation Learning

  • Captured screenshots while manually playing and mapped each frame to the corresponding key press to build a labeled dataset.
  • Preprocessed images (resize, grayscale, normalize) and trained a CNN model to predict actions from visual input.
  • Deployed the model live: used AutoHotkey to simulate key presses in the emulator based on real-time CNN predictions.

Technical Highlight 2 — Grid Map Navigation

  • Captured the emulator window in real time using pyautogui and overlaid a grid (tile size 48) to structure the game space.
  • Identified accessible vs. inaccessible tiles via HSV/RGB color detection (floor RGB range: [90,120,100]–[203,187,168]), detected exits by color (green), and located the character position (blue marker).
  • Iterated through grayscale, RGB, and HSV detection approaches to handle noisy in-game textures; addressed margin noise in exit recognition and hat noise in character detection.
  • Used Q-learning to train a navigation agent on the generated grid map.

Future Work

  • Expand training data to cover more game areas and add temporal memory for tracking movement history.
  • Explore Reinforcement Learning for decision-making and evaluate combining both methods.

📎 ClipME — Website Review Platform

Team Project · Full-Stack Web Development | 2025

Project Overview

ClipME is a social media review platform where users discover, rate, and organize websites and tools based on real experiences from people they actually know. Unlike Reddit, Instagram, or X — where recommendations come from strangers — ClipME focuses on trusted, community-based discovery. Users can follow others, curate a personalized review feed, and vote on reviews to surface the most useful content.

Technical Highlight 1 — Full-Stack Architecture

  • Built a Django REST Framework backend with a clean RESTful API covering authentication, reviews, users, websites, categories, and voting — over 20 endpoints.
  • Implemented JWT token authentication (access + refresh tokens) using SimpleJWT, with role-based access control for RegisteredUser, Moderator, and Admin.
  • Deployed the database on AWS RDS (PostgreSQL) with local SQLite fallback for development; toggled via environment variable.

Technical Highlight 2 — Social Feed & Interaction System

  • Designed a follow/unfollow system where each user's feed dynamically shows reviews only from people they follow, separate from a global Trending feed ranked by upvotes.
  • Built an upvote/downvote voting system on reviews, with filtering and sorting by newest, oldest, or vote count, and date-range filters (7 days, 30 days).
  • Developed the frontend in React 19 + Vite with React Router DOM for client-side routing, Axios for API calls, and Bootstrap for responsive UI.
