💬 YouTube Comment Sentiment Analysis

Research Project · NLP | 2025

Project Overview

Analyzed viewer comment data across English-learning YouTube channels to understand what draws ESL audiences and whether the content supports language acquisition.

Technical Highlights

  • Collected and processed comments from 305+ videos via the YouTube Data API, extracting structured fields into a per-channel dataframe.
  • Built an automated batch submission pipeline using the OpenAI Batch API — chunked comments into 400-comment batches, controlled concurrency with MAX_CONCURRENT = 1 to stay within token limits, and tracked job status via a tracking_log.csv.
  • Implemented failure recovery: failed jobs can be re-queued by flipping their log status from fail → pending and re-running the script, with no manual re-work required.
  • Applied bigram frequency analysis after stop-word removal to surface recurring viewer themes per channel.
  • Used multi-label classification (OpenAI API, ~20 label categories) to categorize comments into thematic clusters; aggregated results using like-weighting (log transform) and normalization.
  • Scored comments along 4 learning dimensions using keyword stem matching: Listening, Speaking/Pronunciation, Vocabulary/Grammar, and Confidence/Motivation.

Graduate Research Project · NLP × Computational Chemistry | 2024–2025

Project Overview

Technical Highlights

  • Textual embedding pipeline: Prompted Llama-3.1-8B to generate per-molecule pharmacological descriptions from SMILES strings and molecular metrics, then encoded outputs into dense vectors using BGE-M3.

  • Pipeline integration: Coordinated parallel textual and structural feature generation into a unified input for the regression model.

🎮 Pokémon AI Agent

Team Project · Computer Vision & Reinforcement Learning | 2025

Project Overview

Built an AI agent that plays Pokémon Gold (Game Boy Color, 1999) by interpreting visual input from a Game Boy emulator and making real-time decisions — no access to the game's internal state, only screen pixels. The project explored two independent methods: a CNN-based imitation learning agent and a grid-map navigation agent using Q-learning.

Technical Highlight 1 — CNN Imitation Learning

  • Captured screenshots while manually playing and mapped each frame to the corresponding key press to build a labeled dataset.
  • Preprocessed images (resize, grayscale, normalize) and trained a CNN model to predict actions from visual input.
  • Deployed the model live: used AutoHotkey to simulate key presses in the emulator based on real-time CNN predictions.

Technical Highlight 2 — Grid Map Navigation

  • Captured the emulator window in real time using pyautogui and overlaid a grid (tile size 48) to structure the game space.
  • Identified accessible vs. inaccessible tiles via HSV/RGB color detection (floor RGB range: [90,120,100]–[203,187,168]), detected exits by color (green), and located the character position (blue marker).
  • Iterated through grayscale, RGB, and HSV detection approaches to handle noisy in-game textures; addressed margin noise in exit recognition and hat noise in character detection.
  • Used Q-learning to train a navigation agent on the generated grid map.

Future Work

  • Expand training data to cover more game areas and add temporal memory for tracking movement history.
  • Explore Reinforcement Learning for decision-making and evaluate combining both methods.

📎 ClipME — Website Review Platform

Team Project · Full-Stack Web Development | 2025

Project Overview

ClipME is a social media review platform where users discover, rate, and organize websites and tools based on real experiences from people they actually know. Unlike Reddit, Instagram, or X — where recommendations come from strangers — ClipME focuses on trusted, community-based discovery. Users can follow others, curate a personalized review feed, and vote on reviews to surface the most useful content.

Technical Highlight 1 — Full-Stack Architecture

  • Built a Django REST Framework backend with a clean RESTful API covering authentication, reviews, users, websites, categories, and voting — over 20 endpoints.
  • Implemented JWT token authentication (access + refresh tokens) using SimpleJWT, with role-based access control for RegisteredUser, Moderator, and Admin.
  • Deployed the database on AWS RDS (PostgreSQL) with local SQLite fallback for development; toggled via environment variable.

Technical Highlight 2 — Social Feed & Interaction System

  • Designed a follow/unfollow system where each user's feed dynamically shows reviews only from people they follow, separate from a global Trending feed ranked by upvotes.
  • Built an upvote/downvote voting system on reviews, with filtering and sorting by newest, oldest, or vote count, and date-range filters (7 days, 30 days).
  • Developed the frontend in React 19 + Vite with React Router DOM for client-side routing, Axios for API calls, and Bootstrap for responsive UI.

🛡️ Cyberbullying Tweet Detection

Team Project · Machine Learning for Cybersecurity | 2025

Project Overview

This project builds a multi-class tweet classifier to detect and categorize cyberbullying on social media. Using the CBTweets dataset (~47K labeled tweets), we trained and compared four models — Decision Tree, Linear SVM, Logistic Regression, and a CNN — to classify tweets into six categories: age-based, gender-based, ethnicity-based, religion-based, other cyberbullying, and non-cyberbullying.

Technical Highlight 1 — Classical ML Pipeline

  • Applied a text preprocessing pipeline to strip URLs, @mentions, hashtags, and non-alphabetic characters, followed by lowercasing and tokenization.
  • Used TF-IDF vectorization (with English stop-word removal) as input features for Decision Tree, Linear SVM, and Logistic Regression models.
  • Tuned hyperparameters via 5-fold GridSearchCV; Logistic Regression (L1, C=1) achieved the best test accuracy at 82.5%.

Technical Highlight 2 — CNN with Word Embeddings

  • Built a CNN classifier using a Keras Embedding layer (vocab size 5,000, dim 128) followed by Conv1D, GlobalMaxPooling1D, Dropout (0.5), and a softmax Dense layer — no manual feature engineering.
  • Filtered tweets exceeding 200 tokens and padded sequences to a fixed length for batch training over 5 epochs.
  • CNN matched classical models at ~81.8% test accuracy, demonstrating that learned embeddings are competitive with TF-IDF on short social media text.

🦠 CNN Malware Family Classification

Graduate Coursework · Machine Learning for Cybersecurity (MIT 3.681) | 2025
25-class image classification · 9,294 malware binary visualizations

Project Overview

Designed and trained a CNN to classify malware binaries into 25 distinct family categories. Malware executables were pre-converted into grayscale images — a standard visualization technique that exposes structural byte patterns — and fed into an image classification pipeline built with TensorFlow/Keras.

Technical Highlights

  • Loaded and split 7,423 training images into an 80/20 train-validation split using tf.keras.preprocessing.image_dataset_from_directory with a fixed random seed for reproducibility.
  • Built a sequential CNN with three Conv2D + MaxPooling blocks (32→64→128 filters), followed by a Dense(256) layer with 0.5 Dropout for regularization — 44.9M parameters total.
  • Tuned hyperparameters including learning rate (Adam, lr=0.0005), number of epochs (30), filter sizes, and dense layer width to push validation accuracy above the 95% target.
  • Monitored training with per-epoch accuracy and loss curves to detect overfitting; validation accuracy plateaued near 97% while training continued to improve, indicating mild overfitting.
  • Evaluated the final model on a held-out test set of 1,871 images across all 25 classes.

Key Results

The model achieved 97.3% validation accuracy and 96.9% test accuracy, exceeding the 95% course benchmark. Validation accuracy was marginally higher than test accuracy, consistent with the model having seen the validation distribution implicitly through early stopping decisions during tuning.

📊 Crime & Socioeconomic Factor Explorer

Interactive dashboard · 1,000-region dataset · 6 socioeconomic variables

Project Overview

Developed an interactive R Shiny web application to investigate how socioeconomic conditions relate to crime rates across 1,000 synthetic regions. The dashboard allows users to freely explore pairwise relationships between variables including Crime Rate, Education Level, Employment Rate, Median Income, Poverty Rate, and Population Density — enabling data-driven hypothesis exploration without any coding required.

Technical Highlights

  • Built a fully reactive Shiny UI with dynamic axis variable selectors, letting users choose any two variables for the scatter plot at runtime using selectInput and .data[[]] tidy evaluation.
  • Added optional OLS regression line overlay via geom_smooth(method = "lm"), toggled by a checkbox — letting users visually assess linear trends on demand.
  • Exposed interactive visual controls (point size and alpha sliders) to help users manage overplotting when exploring dense data distributions.
  • Computed and displayed live Pearson correlation between any selected variable pair using cor() with pairwise-complete observations.
  • Rendered per-variable summary statistics (min, max, quartiles, mean) inline within the sidebar for quick distributional awareness.
  • Styled the app with a gradient background using shinyWidgets::setBackgroundColor and a clean theme_minimal() plot aesthetic.

Key Results

The dashboard revealed intuitive but quantifiable relationships in the dataset — notably a negative correlation between median income and crime rate, and a positive correlation between poverty rate and crime rate — consistent with well-established criminological patterns. The app made these patterns immediately legible to non-technical users through interactive exploration rather than static charts.

💧 Dissolved Oxygen & Water Quality Analysis

Graduate Coursework · Statistical Methods | 2024
Back Bay National Wildlife Refuge, Virginia Beach · 1994–2019

Project Overview

Observational regression study examining which water quality variables — water temperature, salinity, year, and pH — are associated with dissolved oxygen (DO) levels at a federally monitored wildlife refuge. Built and compared full and reduced linear regression models using bi-weekly sampling data spanning 25 years.

Technical Highlights

  • Built a full multiple regression model with four predictors (water temperature, salinity, year, pH); identified pH as non-significant (p = 0.197, two-sided) and removed it via model selection.
  • Compared full vs. reduced models using AIC, BIC, adjusted R², and extra-sum-of-squares F-test — reduced model showed lower BIC (5020.66 vs. 5026.08) with equivalent explanatory power (adj. R² = 0.324).
  • Validated model assumptions via residual plots and QQ-plots; interpreted failed Shapiro-Wilk tests (p = 0.017 / 0.023) in the context of large sample size (n = 1,338).
  • Applied correlation analysis across all quantitative variables; identified a moderate negative correlation between DO and water temperature (r = −0.502).

Key Finding

Water temperature, salinity, and year are each significantly associated with dissolved oxygen levels (p < 0.0001). pH showed no significant association. As an observational study, no causal inference can be drawn.
