Can a Model Learn What Counts as Cyberbullying?

Multi-Class Tweet Classification Across 6 Harassment Categories

Date 2025
Dataset ~47K tweets · 6 classes
Best Accuracy 82.5% (Logistic Regression)
Python scikit-learn TensorFlow NLP TF-IDF CNN GridSearchCV
Abstract

Cyberbullying on social media is pervasive but difficult to detect automatically — the same words carry different intent depending on context, target, and tone. This project tackles the problem as a multi-class classification task using the CBTweets dataset (~47K labeled tweets across 6 categories). We benchmark four models — Decision Tree, Linear SVM, Logistic Regression, and a CNN — and find that while identity-based harassment (age, ethnicity, religion) is learnable with high confidence, ambiguous categories like other_cyberbullying and not_cyberbullying remain challenging for all approaches.

1. Dataset

The CBTweets dataset contains 47,692 tweets pre-labeled into six cyberbullying categories. The class distribution is roughly balanced across the five bullying types, making it suitable for direct multi-class classification without oversampling.

Category Description
age Harassment targeting someone's age group
ethnicity Racial or ethnic slurs and attacks
gender Sexist or gender-based abuse
religion Attacks targeting religious identity
other_cyberbullying Harassment not fitting the above types
not_cyberbullying Negative or edgy content that is not harassment
47,692 Total tweets
6 Classes
~19 Median tweet length (tokens)
4 Models compared

A small number of tweets exceeded 200 tokens (outliers from thread-style content) and were removed before model training, leaving 47,397 clean samples. Tweet length followed a right-skewed distribution with Q1=12, median=19, and Q3=32 tokens — compact text that limits context for classifiers.

2. Methods

2.1 Text Preprocessing

All tweets were cleaned using a unified pipeline before being passed to any model. The goal was to retain only semantically meaningful word tokens.

URL removal @mention stripping Hashtag removal Non-alphabetic filtering Lowercasing Length filtering (≤ 200 tokens)

2.2 Classical ML — TF-IDF + Classifiers

For Decision Tree, Linear SVM, and Logistic Regression, cleaned tweets were vectorized using TF-IDF with English stop-word removal, producing sparse feature matrices. Each model was tuned via 5-fold GridSearchCV on 80% of the data, evaluated on the held-out 20%.

TF-IDF vectorization 5-fold GridSearchCV 80 / 20 train-test split random_state=42

2.3 Deep Learning — CNN

The CNN model used a learned Embedding layer (vocab size 5,000, dimension 128) instead of hand-crafted TF-IDF features. Sequences were padded to a fixed length of 200 tokens. The architecture: Embedding → Conv1D (128 filters, kernel 5) → GlobalMaxPooling1D → Dropout (0.5) → Dense (64, ReLU) → Dense (6, softmax).

Keras Tokenizer (vocab=5,000) Embedding dim=128 Conv1D + GlobalMaxPooling Dropout 0.5 Adam optimizer 5 epochs

3. Results

CNN Embedding + Conv1D, 5 epochs — 81.8%
Identity-based categories are highly learnable Finding 1

Age, ethnicity, and religion categories achieved F1 scores of 92–97% across all models. These categories contain consistent lexical signals — slurs, group identifiers, and attack patterns — that are reliably picked up by both TF-IDF weights and convolutional filters.

Ambiguous categories remain the bottleneck Finding 2

not_cyberbullying and other_cyberbullying had the lowest F1 scores across all models (40–66%). The boundary between edgy-but-acceptable language and genuine harassment is ambiguous even to human annotators, and this ambiguity propagates directly into model confusion — the two classes are consistently misclassified into each other.

CNN matches classical ML without feature engineering Finding 3

The CNN achieved 81.8% test accuracy — comparable to Linear SVM and Logistic Regression — without any TF-IDF vectorization or stop-word removal. This suggests that for short social media text, learned word embeddings can substitute for hand-crafted sparse features, and that the performance ceiling for this task is largely determined by label ambiguity rather than model capacity.

4. Limitations & Takeaways

All models struggle with the not_cyberbullying / other_cyberbullying boundary — a problem rooted in annotation ambiguity rather than model weakness. The dataset is also Twitter-specific; models trained here may not generalize to platforms with different norms (Reddit, Discord, TikTok comments). Extending to contextual models (e.g., BERT fine-tuned on tweet data) would likely improve performance on the ambiguous categories.

  • Logistic Regression with L1 regularization narrowly outperforms all other models — a strong, interpretable baseline for short-text classification.
  • CNN performance is on par with classical ML, validating learned embeddings as a viable alternative to TF-IDF on this scale.
  • The hard ceiling (~82%) is driven by label ambiguity in other_cyberbullying and not_cyberbullying, not model expressiveness.
  • Word clouds per category reveal distinct vocabulary clusters — slurs for ethnicity/religion, gendered terms for gender — confirming that lexical features are genuinely informative for identity-based categories.
  • Future work: fine-tune a Twitter-pretrained BERT (e.g., BERTweet) to capture sarcasm, code-switching, and context that n-gram features miss.
